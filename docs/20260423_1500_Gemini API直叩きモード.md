# Gemini API 直叩きモード（`AI_CLI_PROVIDER=gemini-api`）

Gemini CLI 経由ではなく、**Google AI Studio の Gemini API を直接呼び出す**モードです。Admin の AI 情報収集・議案ステータスチェックで利用できます。

本ドキュメントは [Fork手順](./20260422_1000_Fork手順.md) / [Windsurf+GeminiCLIでのFork手順](./20260422_1400_Windsurf%2BGeminiCLIでのFork手順.md) の補足です。

---

## なぜこのモードがあるのか

Gemini CLI 経由だと、Google アカウントのサインインベースで **無料枠の RPM（1分あたりのリクエスト数）が非常に厳しい** という問題があります。

| プロバイダ | 既定モデル | 無料枠 RPM | 無料枠 RPD |
| --- | --- | --- | --- |
| Gemini CLI (`gemini`) | 2.5 Pro | 5 | 100 |
| Gemini CLI (`gemini` + `GEMINI_MODEL=gemini-2.5-flash`) | 2.5 Flash | 10 | 250 |
| **Gemini API 直叩き (`gemini-api`)** | **2.5 Flash-Lite** | **15** | **1,000** |

AI 情報収集は Web ページを何度も fetch するため、**1 分あたり数リクエスト走らせるだけで CLI 側はすぐ「You have exhausted your capacity on this model」** に達します。直叩きモードなら無料枠内で十分なワークロードが流せます。

また `gemini-api` モードは **CLI のローカルインストール不要**・**サブプロセス起動オーバーヘッドなし**・**Google OAuth サインイン不要** といった副次メリットもあります。

## セットアップ

### 1. API キーを取得

[Google AI Studio](https://aistudio.google.com/app/apikey) にアクセスし、Google アカウントでサインインして **「Create API key」** を押します。発行されたキーをコピーしておきます。

無料枠で利用する場合は「Free of charge」プランで問題ありません（2026-04 時点）。

### 2. `.env` を設定

```dotenv
AI_CLI_PROVIDER=gemini-api

# Google AI Studio の API キー（必須）
GOOGLE_AI_API_KEY=AIzaSy...

# 使用モデル（省略時 gemini-2.5-flash-lite）
# 無料枠の制限が最も緩いので既定推奨。精度優先なら gemini-2.5-flash も可（RPD は少し下がる）。
GEMINI_MODEL=gemini-2.5-flash-lite
```

### 3. Admin を再起動

```bash
pnpm dev
```

`.env` の変更は Admin の再起動で反映されます。

## 動作の違い

| 観点 | `gemini` CLI | `gemini-api` 直叩き |
| --- | --- | --- |
| プロセス | Node から子プロセスで `gemini` コマンドを spawn | Node 内で直接 Gemini API を呼び出し |
| 認証 | CLI 側で Google OAuth サインイン | API キー（`GOOGLE_AI_API_KEY`） |
| Web アクセス手段 | `WebFetch` / `WebSearch` ツール（CLI が保持） | `googleSearch` + `urlContext` grounding tool |
| プロンプト | 「Write ツールで結果をファイルに書き込んで」指示 | 「応答本文に JSON を出力して」指示 |
| タイムアウト | 30分（CLI プロセス強制終了） | 30分（AbortSignal） |
| 使用制限時のエラー | stderr の usage limit パターンを検出 | `ApiError.status === 429` を検出 |

プロンプトはプロバイダに応じて `build-prompt.ts` / `build-status-check-prompt.ts` が自動で切り替えます。呼び出し側のコードは一切変わりません。

## トラブルシューティング

| 症状 | 対処 |
| --- | --- |
| `GOOGLE_AI_API_KEY が未設定です` | `.env` に API キーを貼り付けて Admin 再起動 |
| `Gemini API の使用制限に達しました` | 無料枠の RPD 上限到達。`GEMINI_MODEL=gemini-2.5-flash-lite` に下げる、または翌日まで待つ |
| `Gemini API が空の応答を返しました` | プロンプトが安全性フィルタに引っかかっている可能性。`siteConfig.councilBillsDetailUrl` に不正な値が入っていないか確認 |
| `urlContext` で期待ページが取得できない | Gemini の `urlContext` は 1 回の呼び出しで最大 20 URL まで。対象 URL が動的レンダリングやログインが必要な場合は取得できません |
| 実行がタイムアウトする | `googleSearch` と `urlContext` を組み合わせた grounding 呼び出しは通常 1〜3 分で完了します。30 分超えは異常値なので `GEMINI_MODEL` を確認 |

## 既存ユーザーが切り替えるときの注意

- `gemini` CLI からの切り替えは `.env` 変更のみで OK。**コードやデータの変更は不要**です。
- API キーは有料プランにアップグレードした場合も同じキーがそのまま使えます。課金プランに上げれば RPM/RPD 上限が大きく引き上がります。
- `AI_CLI_PROVIDER` を `gemini` に戻せばいつでも CLI 経由に戻せます。
