# ローカル環境起動手順

**作成日:** 2026-02-19

---

## 前提条件（初回のみ）

以下がインストール済みであること：

- **Docker Desktop** — Supabase のローカル DB に必要
- **Node.js** (v20+) & **pnpm** — `npm install -g pnpm` でインストール可
- **Supabase CLI** — `brew install supabase/tap/supabase` または `npm install -g supabase`

---

## 起動手順（PC 再起動後の毎回の手順）

### Step 1: Docker Desktop を起動する

アプリケーションフォルダから **Docker Desktop** を起動し、メニューバーのクジラアイコンが「Running」になるまで待つ。

> Docker が起動していないと Supabase が起動できません。

---

### Step 2: プロジェクトディレクトリに移動

```bash
cd ~/MIRAI/mirai-gikai-kawasaki
```

---

### Step 3: Supabase（ローカル DB）を起動する

```bash
npx supabase start
```

起動完了後、以下のような出力が表示される：

```
API URL: http://127.0.0.1:54421
DB URL: postgresql://postgres:postgres@127.0.0.1:54432/postgres
Studio URL: http://127.0.0.1:54423
...
```

> 初回は Docker イメージのダウンロードが走るため数分かかることがある。
> 2回目以降は数十秒で起動する。

---

### Step 4: 環境変数ファイルを確認する

`.env` ファイルが存在しない場合は `.env.example` からコピーして作成する。
ローカル開発では基本的に `.env.example` の値をそのまま使える。

```bash
# .env が存在しない場合のみ実行
cp .env.example .env
```

---

### Step 5: 開発サーバーを起動する

```bash
pnpm dev
```

以下の3サーバーが並列起動する：

| サービス | URL |
|----------|-----|
| Web（公開サイト） | http://localhost:3000 |
| Admin（管理画面） | http://localhost:3001 |

> 初回起動時は Next.js のビルドに少し時間がかかる。
> ターミナルに `✓ Ready` と表示されたらアクセス可能。

## DB の中身の確認手段

### 方法1: Supabase Studio（GUI）

ブラウザで http://127.0.0.1:54423 を開く。
テーブルの内容を GUI で閲覧・編集できる。

主要テーブル：

| テーブル | 内容 |
|----------|------|
| `council_sessions` | 定例会（会期） |
| `bills` | 議案 |
| `committees` | 委員会 |
| `factions` | 会派 |
| `faction_stances` | 会派の議案に対する見解 |
| `tags` | タグ |

### 方法2: psql（CLI）

```bash
psql "postgresql://postgres:postgres@127.0.0.1:54432/postgres"
```

接続後の基本コマンド：

```sql
-- テーブル一覧
\dt

-- 議案一覧
SELECT id, title, status FROM bills;

-- 定例会一覧
SELECT name, slug, is_active FROM council_sessions;

-- 会派見解一覧
SELECT b.title, f.display_name, fs.type
FROM faction_stances fs
JOIN bills b ON fs.bill_id = b.id
JOIN factions f ON fs.faction_id = f.id;

-- 終了
\q
```

## 情報の更新手順

### シードデータの再投入（DB を初期状態にリセット）

```bash
pnpm db:reset
```

> これは DB の**全データを削除してから**マイグレーションを再実行し、シードデータを投入する。
> 開発中に DB が壊れたときや、データを初期状態に戻したいときに使う。

### シードデータのみ再投入（リセットなし）

```bash
pnpm seed
```

> マイグレーションは走らず、seed スクリプトのデータのみ再投入する。

### Admin 画面から議案・定例会を手動更新

http://localhost:3001 にログインして GUI から編集できる。
変更は即時 DB に反映される。

### テーブルスキーマを変更したとき

```bash
# マイグレーションを実行 & 型ファイルを再生成
pnpm db:migrate
```

---

## 終了手順

```bash
# 開発サーバーを停止（Ctrl+C）
# その後、Supabase を停止
npx supabase stop
```

> Supabase を停止しなくても Docker Desktop を終了すれば DB は止まる。
> ただし `supabase stop` を使うと次回起動が確実に速い。

## トラブルシューティング

| 症状 | 対処 |
|------|------|
| `supabase start` が失敗する | Docker Desktop が起動しているか確認 |
| localhost:3000 が表示されない | `pnpm dev` が起動中か確認。ターミナルのエラーを確認 |
| DB に接続できない | `npx supabase status` でサービス状態を確認 |
| ページは表示されるがデータが空 | `pnpm seed` を実行してシードデータを投入 |
| 型エラーが出る | `pnpm db:types:gen` で型ファイルを再生成 |
