# 別地域向け Fork 手順書

> このガイドは、川崎市議会版（みらい議会＠川崎市）を別の地方議会向けに Fork する際の手順をまとめたものです。
> 神奈川県議会版（2026-03 Fork 実施）の経験を反映しています。

---

## 0. 重要事項
- このForkプロジェクト作成後、公式リポジトリに下記のForkガイドラインが用意されています。  
  作成当時より非公式である旨記載する等考慮しておりましたが、ドキュメント等には未対応の箇所がある可能性があります。
- 不特定多数の閲覧が可能なリリースをされる場合、必ず最新のガイドラインを参照の上、準拠しているか確認お願いします。
  - [FORK_GUIDELINES.md](../../FORK_GUIDELINES.md)

## 1. はじめに

本リポジトリはオープンソースで公開されており、他の地方議会向けにカスタマイズして利用できます。
地域固有の情報は `site.config.ts` に集約されているため、**最小限の変更でForkが可能**です。

ただし `site.config.ts` 以外にもハードコードされた地域参照（UI テキスト・コメント・シードデータ等）があるため、セクション 3 の確認が必須です。

### 3-3. 「市議会」「市民」等の市レベル固有表現（県議会等にForkする場合）

市議会→県議会のように**議会レベルが変わる場合**は、「川崎」以外に「市議会」「市民」等の表現も置換が必要です。

```bash
# 「市議会」の検索（「県議会」等に置換が必要な箇所を特定）
grep -r "市議会" web/src admin/src --include="*.ts" --include="*.tsx" | grep -v site.config | grep -v node_modules

# 「市民」の検索（UI表示テキストは確認が必要）
grep -r "市民" web/src admin/src --include="*.ts" --include="*.tsx" | grep -v node_modules
```

| 表現 | 対応方針 |
|---|---|
| `市議会` → UI表示テキスト | `siteConfig.councilName` を使用するか、「議会」に汎用化 |
| `市議会` → admin UIラベル（「市議会URL」等） | 「議会URL」に汎用化 |
| `市民` → AIプロンプト・インタビュー設定内 | 広義の「市民（citizen）」として残してよい場合が多い |
| `市民` → UI表示テキスト（「市民の皆さまが」等） | 「皆さまが」に汎用化するか、地域呼称に置換 |

### 7-1. OpenAI API キーの取得

議案サムネイル画像の自動生成（DALL-E）に OpenAI API キーが必要です。

1. [OpenAI Platform](https://platform.openai.com/) でアカウントを作成
2. API Keys ページで新しいキーを発行
3. `.env` の `OPENAI_API_KEY` に設定
4. DALL-E の利用には OpenAI アカウントへのクレジット追加（従量課金）が必要です

## 9. Langfuse プロンプト設定

AIチャット・インタビュー機能では Langfuse でプロンプトを管理しています。

1. Langfuse のプロジェクトを作成し、環境変数を設定
2. 以下のプロンプトを議会・地域情報に合わせて更新：

| プロンプト名 | 用途 | 地域カスタマイズ |
|---|---|---|
| `top-chat-system` | トップページAIチャット | 議会名・地域名の反映 |
| `bill-chat-system-normal` | 議案詳細チャット（通常難易度） | 議会名の反映 |
| `bill-chat-system-easy` | 議案詳細チャット（やさしい） | 同上 |
| `bill-chat-system-hard` | 議案詳細チャット（詳しい） | 同上 |

3. 更新後は管理画面で動作確認

> **注意**: インタビュー機能のプロンプトは現在コード内で動的生成されており、Langfuse では管理されていない。地域固有の文言がある場合はコード側も確認が必要。

## 10. クロスプラットフォーム対応（Windows / Mac）

本リポジトリは Mac・Windows 両方での開発に対応しています。以下の点に注意してください。

### 10-1. devスクリプトのポート指定

`web/package.json` と `admin/package.json` の dev スクリプトは、Bash の `${VAR:-default}` 構文ではなく Node.js の `process.env` 経由でポートを読み取る方式になっています。**この方式は Windows でも動作するため、変更不要です。**

もし独自にスクリプトを追加する場合は、Bash 構文（`${VAR:-default}`）を避け、Node.js または `cross-env` 等のクロスプラットフォームツールを使用してください。

### 10-2. Claude CLI（AI情報収集機能）の Windows 対応

`admin/src/features/ai-collection/server/utils/execute-claude.ts` に Windows 対応が組み込まれています：

| 対応項目 | 内容 |
|---|---|
| CLI パス解決 | Windows では `%APPDATA%/npm/claude.cmd` を自動検出 |
| shell オプション | Windows では `.cmd` ラッパー実行のため `shell: true` を自動設定 |
| プロンプト受け渡し | コマンドライン引数ではなく stdin 経由（Windows のコマンド長制限を回避） |

Windows で AI 情報収集機能を使う場合は、`CLAUDE_CLI_PATH` 環境変数に Claude CLI のフルパスを設定するか、npm グローバルインストール（`npm install -g @anthropic-ai/claude-code`）を行ってください。

### 10-3. Supabase CLI

`npx supabase start` は Docker を使用します。Windows の場合は WSL2 + Docker Desktop が必要です。Mac では Docker Desktop または OrbStack 等を使用してください。

### 10-4. 改行コード

Biome が LF を強制するため、Windows で `git clone` する際は改行コードの自動変換を無効にすることを推奨します：

```bash
git config core.autocrlf input
```
